Brussels Pedestrian Network

Outil d'analyse et de QA pour les contributeurs OpenStreetMap
Carte Statistiques Évolution Méthodologie & QA

Le Brussels Pedestrian Network n'est pas un simple outil de rendu, mais un instrument d'analyse conçu pour aider la communauté OpenStreetMap à identifier et corriger les lacunes du réseau piéton bruxellois.

Objectif Mappeur : Utiliser la simulation de flux pour prioriser le mapping là où l'impact est le plus fort, et utiliser les couches de QA pour détecter les erreurs de topologie ou les tags manquants.

1. Interprétation des Tags et Coûts de Routage

Le moteur de routage (igraph) utilise des coefficients de pénalité basés sur les tags OSM pour simuler l'attractivité des chemins. Un coût élevé détourne les flux simulés vers d'autres axes.

Tag HighwayCoût (Poids)Note
pedestrian, footway1.0Infrastructure idéale.
living_street, path1.2Mixte ou informel.
steps1.5Obstacle physique (poussettes, PMR).
residential, service2.0Chaussée partagée.
tertiary3.5Trafic plus important.
primary8.0Axe majeur, très dissuasif sans trottoir.

Influence du tag sidewalk

La présence de trottoirs sur les routes (residential, tertiary, etc.) modifie drastiquement le coût :

  • sidewalk=both ou separate : Coût réduit (proche de 1.0).
  • sidewalk=none : Pénalité forte (coût x 0.3 d'efficacité).
  • Tag absent : Pénalité modérée (0.5). C'est ici que le mapping a le plus d'impact !

2. Couches de Contrôle Qualité (QA)

L'outil propose plusieurs couches spécifiques pour orienter vos sessions d'édition :

Trottoirs unilatéraux (Sidewalk Gaps)

Une analyse spatiale compare l'axe central de la route avec les highway=footway parallèles. Si un seul côté dispose d'un trottoir mappé séparément, le segment est marqué. Cela aide à compléter le mapping "micro-mapping" de façon systématique.

Flux sur route (Forced Segments)

Cette couche affiche les segments de route où la simulation prédit un flux piéton important alors qu'aucune infrastructure dédiée (trottoir mappé ou tag sidewalk) n'est renseignée. Action : Vérifier sur le terrain ou via imagerie si un trottoir existe et ajouter le tag ou le chemin.

Traversées manquantes

Le pipeline détecte les nœuds highway=crossing qui ne sont connectés à aucun highway=footway avec footway=crossing. Ces points rompent la connectivité du graphe pour les analyses de marchabilité.

3. Score de Complétude

Le tableau de bord des statistiques affiche un score de complétude basé sur :

  • Documentation des routes : Ratio des segments routiers possédant un tag sidewalk explicite.
  • Connectivité : Ratio de nœuds de traversée connectés au réseau piéton.
  • Précision (Separate) : Part des trottoirs mappés comme des objets géométriques distincts (plus précis pour le guidage PMR).
  • Attributs d'accessibilité : Présence des tags surface, tactile_paving et smoothness.

Comment contribuer ?

Chaque lundi, les données sont rafraîchies à partir d'OpenStreetMap. Vous pouvez :

  1. Identifier une zone rouge (flux sur route) ou un "gap" de trottoir sur la carte.
  2. Cliquer sur le bouton "Éditer dans OSM" (accessible au zoom 16+).
  3. Ajouter les informations manquantes dans iD ou JOSM.
  4. Voir vos changements apparaître lors de la mise à jour hebdomadaire.
